**Issues:**

- No pricing tiers -- `servicePricing.json` does not exist, so `hasOfferCatalog` is empty. The pricingNote text mentions $750-$8,000/month but this is not structured.

**Recommendation:**
1. Change `description` in JSON to start with "Local SEO services..." for schema name alignment.
2. Pass `areaServed` to the schema helper -- add Canada + Edmonton.
3. Create `data/servicePricing.json` with local-seo tiers to enable Offer schema:
   - Starter: $750 CAD/month
   - Multi-Location: $1,500 CAD/month
   - Enterprise: $8,000 CAD/month

### 5b. FAQ Schema
**CRITICAL BUG FOUND:** The local-seo FAQ entries use `"question"` / `"answer"` keys, but the template at line 129 reads `$f['q']` / `$f['a']`:
```php
$faqItems[] = ['question' => $f['q'], 'answer' => $f['a']];
```
Since `$f['q']` is `null` (the JSON key is `"question"`, not `"q"`), the FAQ schema will contain empty strings. Additionally, the FAQ display in the HTML (lines 362, 366) uses `$faq['q']` and `$faq['a']`, so **the FAQs render as blank on the page**.

**Fix required:** Change the JSON keys from `"question"`/`"answer"` to `"q"`/`"a"` to match the template, OR update the template to handle both formats. The schema helper (`tml_schema_faq`) already handles both formats, but the view template does not.

- No `Review` or `AggregateRating` schema despite stats claiming "2000+ Local Rankings".
- No `Article` schema or `author` markup despite author attribution being displayed.

**Priority: CRITICAL** (FAQ bug) / HIGH (missing pricing & area served).
