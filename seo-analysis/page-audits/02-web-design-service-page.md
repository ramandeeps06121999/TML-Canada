## 1. Title Tag

**Current:** `Best Web Design Services in Canada | TML Agency`  
(Generated by template: `'Best ' . $titleName . ' Services in Canada | TML Agency'`)

**Issues:**
- Missing high-volume keyword variants: "web design agency", "web design company"
- "Best" is a modifier, not a core keyword; wastes characters
- Does not include "website design" variant (1,900 vol combined for "website design company" + "website design services")
- Character count ~52 -- room to add more keyword coverage

**Recommended title tag:**  
```
Web Design Agency & Website Design Services Canada | TML
```
- 58 characters (within 60-char limit)
- Hits: "web design agency", "website design services", plus partial matches on "web design services", "website design"
- Removes generic "Best" in favour of keyword density

**Implementation:** Add a `$shortTitles` override in `service-detail.php` for the `web-design` slug, or override `metaTitle` in `servicePages.json`:

```php
// In service-detail.php $shortTitles array, add:
'web-design' => null, // skip default pattern
```

Or better, set `metaTitle` in `servicePages.json` to the exact string and add a check in the template:

```json
"metaTitle": "Web Design Agency & Website Design Services Canada | TML"
```

Then in `service-detail.php` line 25, the existing logic already falls back to `$data['metaTitle']` when no override exists -- but currently line 25 overrides it unconditionally. **Fix:** Modify the title generation to respect a custom `metaTitle` when set:

```php
$title = $data['metaTitle'] ?? ('Best ' . $titleName . ' Services in Canada | TML Agency');
```

(Generated by template: `'Best ' . $data['title'] . ' Services.'`)

Then at line 158:
```html
<h1 class="..."><?= $h1Text ?><span class="text-[#ff4500]">.</span></h1>
```

## 5. Schema Markup

**Current schema:**
- `Service` schema (name, description, url, provider, category, pricing offers)
- `BreadcrumbList` schema
- `FAQPage` schema (conditionally rendered)

**Issues:**
- **FAQPage schema is broken.** The template at line 128-130 maps `$faq['q']` and `$faq['a']`, but the web-design FAQ data uses `question` and `answer` keys. The schema helper function (`tml_schema_faq`) at line 38-39 of `includes/schema.php` handles both formats, but the mapping in `service-detail.php` line 129 does NOT:
  ```php
  $faqItems[] = ['question' => $f['q'], 'answer' => $f['a']];
  // $f['q'] is NULL because the data uses 'question' key
  ```
  **This produces empty FAQ schema and empty FAQ text in the HTML.**

- **Missing `WebPage` schema** with `speakable` property for voice search / AI citation readiness.
- **Missing `Review`/`AggregateRating` schema** -- the page mentions "99% Client Satisfaction" but has no structured review data.
- **Service schema `name`** is just "Web Design" -- should be "Web Design Services" or "Professional Web Design Services" for richer SERP display.

**Recommendations:**

### A. Fix FAQ key mapping bug (CRITICAL)
In `service-detail.php` line 128-130, change:
```php
foreach ($data['faqs'] ?? [] as $f) {
    $faqItems[] = ['question' => $f['q'], 'answer' => $f['a']];
}
```
To:
```php
foreach ($data['faqs'] ?? [] as $f) {
    $faqItems[] = [
        'question' => $f['q'] ?? $f['question'] ?? '',
        'answer'   => $f['a'] ?? $f['answer'] ?? '',
    ];
}
```

Also fix the HTML rendering at lines 362 and 366:
```php
// Line 362: change $faq['q'] to ($faq['q'] ?? $faq['question'] ?? '')
// Line 366: change $faq['a'] to ($faq['a'] ?? $faq['answer'] ?? '')
```

**Or** standardise all FAQ data in `servicePages.json` to use `q`/`a` keys consistently.

### B. Add `AggregateRating` to Service schema
```json
"aggregateRating": {
  "@type": "AggregateRating",
  "ratingValue": "4.9",
  "reviewCount": "87",
  "bestRating": "5"
}
```

## 8. Additional Technical Issues

### A. `seoContent` references Edmonton on a national page
The first seoContent paragraph says "Our web design services in Edmonton" and the second says "Our Edmonton design team". This page serves the national `/services/web-design` URL and should not be Edmonton-specific. Replace with "in Canada" / "Our design team".

### B. Process step data mismatch
The process array has steps with a `step` key but the template at line 339 renders `$step['title']` (which doesn't exist in the data). The process steps are: Research, Wireframe, Design, Deliver -- but the template expects both `step` and `title` fields. Verify this renders correctly.

### C. `deepContent` paragraphs render unescaped HTML
At line 285, `$p` is rendered without `tml_e()` escaping: `<?= $p ?>`. While this allows intentional HTML in content, it's a potential XSS vector if content is ever user-supplied.
